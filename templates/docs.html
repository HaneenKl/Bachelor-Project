Semigroups Documentation

Introduction

This page is designed to allow the user to perform semigroup-related computations using either regular expressions or Arden expressions. The available operations include generating Egg-box diagrams, Cayley graphs, and checking equations based on the elements of the semigroup.

Input Types

Users can choose between two input modes:

  • Arden Expression: This is an input format used for expressing transitions in an automaton.
  • Regular Expression

Arden Expression

In the Arden equation format, each equation defines a transition for a particular state in an automaton. Multiple equations can be entered, one per line. Each equation should consist of terms in the form of a symbol followed by a variable or terminal symbol. The terms of the equation are concatenated by the "+" symbol.
Example:
A = aB + aA + ε
B = bB + cC + c
C = cA

Regular Expression

The input for regular expressions supports the following operations:

  • Concatenation: Characters next to each other are treated as concatenated terms (e.g., "ab").
  • Union: Use the "+" symbol to represent union.
  • Kleene Star: Use "*" to represent the Kleene star operation.

If you wish to treat each entered character as a symbol, check the box below the input field. This will treat each individual character in the input as a separate symbol. For example, entering ab*c without the checkbox checked will treat "ab" and "c" as single characters. The generated language will be {c, abc, ababc, …}.
When the checkbox is checked, each letter will be treated individually as a symbol. So, entering ab*c with the checkbox checked will treat "a", "b", and "c" as separate symbols. The generated language will then be {ac, abc, abbc, …}.

Operations

After selecting a mode and entering the input, the following computations are available:

  • Egg-box Diagram: Generates the Egg-box diagram for the input expression.
  • Left Cayley Graph: Generates the left Cayley graph for the input expression.
  • Right Cayley Graph: Generates the right Cayley graph for the input expression.
  • Equation Check: Allows checking whether multiple equations hold based on the semigroup elements.

Equation Check

The Equation Check section allows users to input multiple equations. Each equation should be entered on a new line.

The input can include exponentiation or omega expressions such as (x y)^w = z^3 x.

Example:

x^3 y = z^w
y^2 z = x
z x = x y^2
(x y)^3 = z^2
z^4 = (y^2 z^3 x)^w

Symbols must be separated by spaces in order to be seen as single symbols.

History

A history panel is available to track the inputs that have been entered. This allows users to see previous modes and inputs. Users can also clear the history if needed.

Conclusion

This page provides an interface to work with semigroup computations. Whether you're working with regular expressions or Arden equations, you can easily generate graphical representations or check the validity of equations.
